1 The resulting sequence of nested structures is called a skiplist, since the upper lists have edges that skip over nodes in the lower lists.
2 The most important concept in studying a skiplist is the notion of a search path. We usually think of a search path as looking for a specific value that may or may not be contained in the bottom list, $L_0$.
2 In this case, we'll consider the search path for 4.5, which is not contained in $L_0$, but if it were, it would be between the values 4 and 5, which both appear in $L_0$.
1 The search path starts at the highest sentinel node and uses a greedy strategy: If the search path can move to the right without overshooting or reaching the value we're looking for, then it does. Otherwise, it moves down.
1 In this case, right step would take us to 9, which is not less than 4.5, so instead, the search path goes down one level, into the list $L_3$.
